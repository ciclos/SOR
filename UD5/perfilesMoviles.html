Cuentas de usuario Vs perfiles de usuario

Un perfil de usuario no es lo mismo que una cuenta de usuario, que se usa para iniciar sesión en Windows. Las preferencias de cada usuario se guardan en un perfil de usuario, (entiendase carpeta) que el Sistema Operativo utiliza para configurar el escritorio cada vez que los usuarios inician sesión. Contiene la configuración para fondos de escritorio, protectores de pantalla, preferencias de puntero, configuración de sonido y otras características. Los perfiles de usuario permiten que se usen sus preferencias personales siempre que inicie sesión en Windows. El perfil de usuario recolecta las opciones de configuración que hacen que el equipo tenga el aspecto y funcione de la manera que usted desee. Cada cuenta de usuario tiene por lo menos un perfil de usuario asociado.

Perfiles moviles

Cuando se trabaja con usuarios de dominio es normal tener perfiles moviles, para atender a los usuarios que usan diferentes equipos pero el mismo usuario en todos ellos. Este tipo de perfiles permite tener siempre el mismo entorno y configuraciones independientemente de la maquina que se use.
Los datos de estos perfiles se almacenan en carpetas compartidas en el servidor, de tal forma que al iniciar sesión esta información se descarga al equipo y al cerrar sesión el perfil se guarda en el servidor. Por tanto lo primero que debemos hacer es crear una carpeta compartida en el servidor, con acceso para todos los usuarios, como se ve a continuación. Fijate que al nombre en la red del directorio se le ha añadido el simbolo $. Esto provoca que el directorio no salga en el listado de objetos compartidos en la red, de tal forma que los usuarios no pueden saber que existe, aunque se puede acceder como a cualquier otro directorio compartido.

Una vez tenemos donde albergar los perfiles, nos vamos a Usuarios y Equipos de Active Directory y dentro de las propiedades de cada usuario ponemos la ruta a la carpeta compartida, seguida de \%username%. Esta ultima cadena se va a sustituir automaticamente por el nombre del usuario, lo que va a simplificar la configuración, pues podemos solo copiar y pegar ese ruta para todos los usuarios.

La primera vez que el usuario inicie sesión en el dominio se creara una carpeta dentro de la carpeta compartida que hemos creado para ello.

Redirección de carpetas

A veces puede ser conveniente redirigir directorios de los usuarios, para que se almacenen en el servirdor, de forma transparente al usuario. A continuación se muestra como es posible redirigir la carpeta Mis Documentos a una carpeta de red compartida en el servidor.
Primero creamos una directiva de grupo en la unidad organizativa correspondiente. Vamos a editar la directiva Redirección de carpetas. Aunque en el ejemplo vamos a redirigir solo Documentos, como se puede observar podemos redirigir cualquier de las carpetas que se indican en la directiva. Editamos las propiedades, seleccionamos basico e indicamos cual es la ruta a la carpeta compartida. Si necesitamos compatibilidad lo podemos indicar en la pestaña configuración.
