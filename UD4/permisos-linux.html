Permisos en Linux

Usuarios en linux

En linux todo archivo y directorio pertenece a un usuario y a un grupo. Cuando se crea un fichero o un directorio se le asigna como propietario el usuario que lo creó y como grupo el de este usuario.

Internamente Linux asigna a cada usuario un número denominado UID que lo identifica mas halla del nombre de usuario.

  • Root o superusuario. Cuenta con todos los privilegios del sistema. Su UID es siempre 0.
  • Usuarios del sistema. No son usuarios en el sentido físico del término, sino que se generan para controlar los servicios a los que pertenecen. Su UID está entre 1 y 1000.
  • Usuarios normales. Son las personas ( u otras entidades ) que se conectarán al sistema para usarlo y que, normalmente, tienen un directorio dentro de home y un UID mayor de 1000.

Permisos

En los Sistemas Unix, la gestión de los permisos que los usuarios y los grupos de usuarios tienen sobre los archivos y las carpetas, se realiza mediante un sencillo esquema de tres tipos de permisos que son:
  • Permiso de lectura
  • Permiso de lectura
  • Permiso de ejecución
A todos los archivos y directorios del sistema se le asigna tres grupos de permisos: propietario, grupo y otros. En realidad estos permisos se gestionan como grupos de tres bits, indicando si el usuario puede leer, escribir o ejecutar, como se ve en la siguiente imagen: Podemos cambiar los permisos por medio del comando:
chmod PERMISOS ARCHIVO.
Para dar los permisos anteriores a un archivo usando chmod escribiríamos lo siguiente: chmod 640 archivo.

Ficheros de configuración

Los principales archivos de control de los usuarios son:
  • /etc/passwd: Contiene una linea por cada usuario. Esta linea tiene el siguiente esquema: login:x:UID:GID:información:directorio_personal:shell_de_inicio. El segundo campo siempre es x, porque la contraseña se guarda en el archivo /etc/shadow.
  • /etc/shadow: Contiene las contraseñas de los usuarios en formato encriptado. Cada línea tiene el siguiente esquema: login:contraseña:ultimo_cambio:min:max:aviso:inactividad:exp
  • /etc/group: Contiene los grupos del sistema. Cada línea tiene el siguiente esquema: nombre:x:GID:usuarios.

profile y bashrc

Son dos archivos de configuración relacionados con la administración de los usuarios y están ubicados en el directorio /etc. El archivo profile se ejecuta cuando qualquier usuario inicia la sesión y bashrc cuando cualquier usuario ejecuta el programa bash.

Aparte de los archivos generales, todos los usuarios contienen en su home una replica oculta de estos archivos para su personalización.

Comandos para gestión de usuarios

  • Creación de usuarios: useradd
  • Modificación de usuarios: usermod
  • Eliminación de usuarios: userdel
  • Creación de grupos: groupadd
  • Modificación de grupos: groupmod
  • Eliminación de grupos: groupdel
  • Saber a que grupos pertenece un usuario: groups
  • Añadir usuarios a un grupo: adduser ( interactivo )
  • Quitar usuarios de un grupo: deluser
  • Cambio de permisos: chmod
  • Cambio de propietario: chown
  • Cambio de grupo: chgrp
  • Cambiar password: chpasswd. (Desde root podemos cambiar el password de cualquier usuario así: echo "usuario:pass" | chpasswd )
